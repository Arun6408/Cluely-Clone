# Software Architecture & IPC Design 🏗

This project runs on the **Tauri v2** framework, leveraging a lightweight WebView frontend (Vanilla JS/HTML/CSS) communicating directly with a high-performance systems-level Rust backend.

## 🦀 Backend (Rust / `src-tauri`)
The backend manages OS bindings, media captures, filesystem reading, and global keyboard shortcuts.
* **`main.rs`**: The main application lifecycle. Hosts the Tauri Command `invoke_handler` mapping frontend requests into Rust threads. Also handles the invisible `.NET Powershell` execution to grab instantaneous screen captures (`take_screenshot`) and the filesystem reader (`read_file_content`) bypassing standard Webview browser limits.
* **`system_audio_minimal.rs`**: The core loopback service. Hooks natively into Windows WASAPI, capturing raw linear PCM output frames from the host speaker, encoding them into base64, and emitting `system-audio-data` payloads asynchronously over Tauri's event bus directly to the frontend socket handler.
* **`windows_capture.rs`**: OS-level configurations (e.g., hiding the window from standard capture streams like OBS or Discord).

## 🌐 Frontend (Vanilla JS / `ui`)
A completely dependency-free (no React/Vue) frontend loop for maximum execution speed, styling, and customization.
* **`index.html`**: The UI skeleton. Contains native CSS targeting chat layouts, tabs, custom Model Dropdowns, and nested `marked.js` Markdown element rendering constraints.
* **`overlay-ui.js`**: The brains of the UI. Interpets STT text chunks into the `Transcription` tab history, builds the AI context strings (injecting your hard drive's Resume/JD), and triggers the auto-switching `Ask AI` view logic when formulating prompts.
* **`prompt_config.js`**: A secure isolation block for maintaining the AI's core identity prompt and file target paths without forcing developers to dig into the application state-loop.
* **`gemini_text_client.js`**: Replaces standard REST wrappers with a pure `fetch()` stream reader, interpreting `Server-Sent Events` (SSE) generated by Gemini's endpoint, yielding partial JSON text chunks dynamically to paint the DOM in real-time.
* **`deepgram_stt_client.js`**: Manages two robust websockets; one piping standard Browser `MediaRecorder` mic audio, and the other ingesting the raw binary PCM streams emitted from the Rust loopback service.
